# Bugs Fragment Template

> Managed document. Must comply with template BUGS_FRAGMENT.template.md.

- Template Version: `1.0`
- Last Updated: `2026-04-02`

## Purpose

Use this fragment to propose changes to the bug record without editing `BUGS.md` directly.

## Required Sections

1. Executive Summary
2. Bug Updates
3. Expected vs Current Behavior
4. Fix and Validation Notes
5. Open Questions
6. Merge Guidance

## Guidance

- Use stable bug IDs when they already exist.
- Be explicit about current behavior, expected behavior, and validation impact.
- Note whether the bug should stay planned, move to implemented history, or trigger follow-up work elsewhere.
