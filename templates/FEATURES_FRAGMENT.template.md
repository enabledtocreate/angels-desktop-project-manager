# Features Fragment Template

> Managed document. Must comply with template FEATURES_FRAGMENT.template.md.

- Template Version: `1.1`
- Last Updated: `2026-04-18`

## Purpose

Use this fragment to propose active feature changes for `FEATURES.md`, especially when new planned work or reorganized planning needs to be merged safely.

## Required Sections

1. Executive Summary
2. Planned Feature Updates
3. Roadmap and Work Item Impact
4. Open Questions
5. Merge Guidance

## Guidance

- Treat `Planned Features` as work still to be implemented.
- Do not add implemented, completed, done, closed, or archived features back into the generated `FEATURES.md` document.
- Historical feature records remain in the database-backed archive/history views.
- Reference stable feature IDs whenever they exist.
- Note whether roadmap phases, PRD content, or other modules need follow-up updates.
