# Features Fragment Template

> Managed document. Must comply with template FEATURES_FRAGMENT.template.md.

- Template Version: `1.0`
- Last Updated: `2026-04-02`

## Purpose

Use this fragment to propose feature changes for `FEATURES.md`, especially when new planned work, implemented work, or reorganized planning needs to be merged safely.

## Required Sections

1. Executive Summary
2. Planned Feature Updates
3. Implemented Feature Updates
4. Roadmap and Work Item Impact
5. Open Questions
6. Merge Guidance

## Guidance

- Treat `Planned Features` as work still to be implemented.
- Treat `Implemented Features` as historical record of completed work.
- Reference stable feature IDs whenever they exist.
- Note whether roadmap phases, PRD content, or other modules need follow-up updates.
